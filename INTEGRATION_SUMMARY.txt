## 🚀 Quick Start (5 Steps)

### Step 1: Run Database Schema
```
1. Go to: https://app.supabase.com
2. SQL Editor → New Query
3. Paste entire content of: supabase-setup.sql
4. Click "Run"
```
✅ Creates: `customers`, `barbers`, `services`, `appointments` tables with RLS

### Step 2: Create Barber Users in Auth
```
1. Go to: Authentication → Users → Add user
2. Create 5 users:
   - liel@ronamar.co.il

   - nevo@ronamar.co.il
   - ron@ronamar.co.il
3. Set passwords for each
4. Copy each User ID (UUID)

```

```
2. Paste the barber INSERT statement from SUPABASE_SETUP.md (Step 3)

4. Click "Run"
```

```
2. Click "הזמן תור עכשיו"
```

## 📊 Database Overview

### Tables Created
```
customers (customer phone + name)
├─ id (UUID)
├─ phone (UNIQUE)
├─ name
└─ timestamps

barbers (linked to Supabase Auth users)
├─ id (UUID from auth.users)
├─ name (UNIQUE)
├─ base_price
├─ bio
└─ timestamps

services (8 barbershop services)
├─ id (serial)
├─ name (UNIQUE)
├─ price_offset or fixed_price
└─ display_order

appointments (the main booking table)
├─ id (UUID)
├─ customer_id → customers
├─ barber_id → barbers
├─ service_id → services
├─ appointment_date
├─ appointment_time (UNIQUE with barber_id, date)
├─ status (confirmed/completed/cancelled)
└─ timestamps
```

## 🔒 Security Features

✅ **Row Level Security** - Each barber sees only their appointments (database enforced)
✅ **UNIQUE Constraints** - No double-booking possible
✅ **Foreign Keys** - Data integrity protected
✅ **RLS Policies** - Multiple levels of access control
✅ **Anon Key Only** - No sensitive keys exposed

## 🎯 Key Points to Remember

1. **Database Schema First** - Run supabase-setup.sql before testing
2. **Auth Users Then Data** - Create Supabase Auth users, then insert into barbers table
3. **RLS is Automatic** - Database enforces access control, not the app
4. **Demo Password** - Use "1234" for barber login (change in production)
5. **SMS is Blocked** - CORS prevents SMS in browser; works fine in production with Edge Functions

## 🐛 Troubleshooting Quick Fixes

| Problem | Solution |
